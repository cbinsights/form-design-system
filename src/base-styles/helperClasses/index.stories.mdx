## Responsive

Select FDS classes can be prefixed to be applied at breakpoints. There are 4 mobile first breakpoints:

<Story name="Responsive">
  <>
    <TableLayout
      isCSS
      headers={['Breakpoint prefix', 'Minimum width']}
      rows={[
        ['`s`', '480px'],
        ['`m`', '768px'],
        ['`l`', '1024px'],
        ['`xl`', '1280px'],
      ]}
    />
  </>
</Story>

The following classes can be prefixed and used responsively:

- [margin & padding](/?path=/docs/style-utilities-style-utilities--margin-and-padding)
- [align](?path=/docs/style-utilities-style-utilities--text-align)
- [alignChild](?path=/docs/style-utilities-style-utilities--child-align)
- [display](/?path=/docs/style-utilities-style-utilities--display)

Example of usage:

```
<div className="padding--top--l l:padding--top--xl" />
```

Guidelines / Notes:

- Don't use `sm:` to target mobile devices
- Use unprefixed utilities to target mobile, and override them at larger breakpoints
- We will add support for more classes to be used responsively in the future.
